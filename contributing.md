# Contributing to SmartBlaster

Thanks for your interest in contributing!

## Hardware Contributions

Hardware is organized by revision:

- `revA` — first working prototype
- `revB` — improvements based on testing
- `revC` — production-ready

Please do not modify files inside an existing revision.  
Instead, create a new revision folder and submit a PR.

## Software Contributions

Software follows a standard branching model:

- `main` — stable
- `dev` — active development
- feature branches — PRs into `dev`

Please include tests for new features.

## Documentation Contributions

Documentation lives in `docs/`.  
Feel free to improve clarity, add diagrams, or expand guides.
